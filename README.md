# Курсовой проект по модулю «Автоматизация тестирования» для профессии «Инженер по тестированию»

Настоящий курсовой проект выполнен в соответствии с [заданием](https://github.com/netology-code/aqa-qamid-diplom).

Целью проекта является автоматизирование тестирования позитивных и негативных сценариев покупки тура на основании [плана]().

По результатам проекта составлена отчётная документация: 
1. [отчёт о тестировании](),
2. [отчёт об автоматизации]().

## Начало работы
Перед работой с проектом следует ознакомиться с перечнем необходимых инструментов (см. п. 2 [Плана]()), установить на компьютере (убедиться в наличии и актуальности) необходимого ПО.
## Запуск проекта
1. Клонировать репозиторий, расположенный по адресу: https://github.com/cedric-ko/qamid-diplom (команда в терминале git clone `git@github.com:cedric-ko/qamid-diplom.git`).
2. Открыть клонированный проект в IDE, например, в IntelliJ IDEA.
3. Запустить приложение Запустить Docker Desktop.
4. В терминале IDE запустить конетейнер командой `docker compose up`, дождаться загрузки контейнера (сообщение ` ready for connections`).
5. Открыть новый терминал и запустить приложение командой `java -jar artifacts/aqa-shop.jar`, дождаться загрузки приложения (сообщение `Started ShopApplication`).
6. Открыть в браузере страницу приложения по адресу: http://localhost:8080/.
7. В приложении DBeaver создать новую базу данных с настройками, расположенными в файле `application.properties` в корне проекта.

## Работа с автотестами
1. Открыть новый терминал в IDE, запустить тесты командой `./gradlew clean test`.
2. По завершении прогона тестов запустить создание отчёта Allure командой в том же терминале `.\gradlew allureServe` (по завершению формирования отчёта он откроется в браузере по умолчанию).

## Завершениеработы с проектом
1. Остановить работу приложения в терминале, где оно было запущено, командой `Ctrl + C`.
2. Остановить работу докера в терминале, где он был запущен, командой `Ctrl + C`, после остановки докера удалить контейнер командой в терминале `docker compose down`.
3. Закрыть приложения (IDE, браузер, DBeaver, Docker Desktop).
