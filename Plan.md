## 1. Перечень автоматизируемых сценариев
   
### Сценарии перехода к формам покупки тура:

**Предусловие**: перед выполнением каждого сценария в браузере должна быть открыта страница тестируемого сервиса "Путешествие дня" https://localhost:8080/

*Ожидаемый результат*: На странице отображается форма для заполнения с заголовком "Оплата по карте" и кнопкой "Продолжить".

#### 2. Переход к форме покупки "Кредит по данным карты":

*Ожидаемый результат*: На странице отображается форма для заполнения с заголовком "Кредит по данным карты" и кнопкой "Продолжить".

### Сценарии заполнения и отправки форм:

**Предусловие**: перед выполнением каждого сценария в браузере должна быть открыта страница тестируемого сервиса "Путешествие дня" http://localhost:8080/ и выбрана одна из форм оплаты (клик по кнопке "Купить" либо "Купить в кредит").

**Важное примечание**: поскольку поля для заполнения, порядок их заполнения и отправки в формах оплаты "Оплата по карте" (кнопка "Купить") и "Кредит по данным карты" (кнопка "Купить в кредит) идентичны, приведённые ниже сценарии будут реализованы для каждой формы оплаты и не повторяются для лаконичности изложения настоящего документа.

**Описание тестовых данных**. Сервис обрабатывает только специальные номера карт, которые даны для тестирования:
  * APPROVED карта — 1111 2222 3333 4444,
  * DECLINED карта — 5555 6666 7777 8888;

**Описание валидных данных**. Для целей реализации следующих сценариев под валидными данными понимаются:

* для поля "Номер карты" валидным является значение, соответствующее номеру карты APPROVED — 1111 2222 3333 4444;
* в качестве валидной даты (поля "Месяц" и "Год") используется текущий месяц текущего года, то есть:
  * для поля "Месяц" "01" для января 2026 года,
  * для поля "Год" "26" для 2026 года,
* для поля "Владелец" валидным является значение, состоящее из латинских букв, пробелов и дефисов. Требования к количеству символов отсутствуют, однако в банковской практике принято считать, что количество символов свыше 30 невозможно эмбоссировать на банковской карте без пересечения с другими полями;
* для поля "CVC/CVV" валидным является значение, состоящее из 3 цифр.

## 2. Перечень используемых инструментов с обоснованием выбора.
   * **IDE**: *IntelliJ IDEA 2025.2.1 (Community Edition)* - интегрированная среда разработки программного обеспечения для многих языков программирования, в частности Java, имеет функции автодополнения и анализа кода, содержит встроенный отладчик, средства для автоматизации сборки проекта, управления зависимостями и интеграции с системами контроля версий.
   * **Платформа разработки**: *Open JDK 11* как инструмент разработки автоматизированных тестов для языке *Java*. Поддерживает работу с фреймворком *JUnit5* для модульного тестирования Java-приложений, совместимым с IDE, системами сборки *Maven* и *Gradle*, работающий в том числе через консоль.
   * **Система автоматической сборки**: *Gradle*, поддерживает выполнение задач компилирования кода, тестирования приложений и отчётности через подключаемые библиотеки *Java*, зависимости, плагины и репозитории.
   * **Фреймворк для автоматизированного тестирования**: *Selenide* - написание тестов веб-приложений на основе *Selenium WebDriver*. Поддерживает работу с веб-браузерами, имеет простой синтаксис.
   * **Подключаемые библиотеки, плагины и зависимости**:
     * *Java Faker* — библиотека для генерации фиктивных данных на Java для задач тестирования, разработки и заполнения данных; 
     * *DbUtils* — библиотека для упрощения работы Java с СУБД; 
     * *Project Lombok* — библиотека для автоматической генерирации шаблонного кода (геттеры, сеттеры, конструкторы, методы equals() и hashCode()) с помощью аннотаций; 
     * *REST Assured* — библиотека для автоматизации тестирования API в RESTful веб-сервисах.
   * **Платформа контейнеризации**: *Docker* — платформа для создания виртуальной среды и контейнеризации приложений. Используется через приложение *Docker Desktop* для доступа к виртуальной БД с тестовыми данными. 
   * **Клиент для работы с БД**: *DBeaver* — удобный инструмент для создания, редактирования и визуализации баз данных (СУБД) с поддержкой скриптов SQL.
   * **Система контроля версий**: *Git* — распределённая система, позволяющая отслеживать и записывать любые изменения в наборе файлов, возвращаться к предыдущим версиям проекта. Веб-сервис для хостинга IT-проектов и их совместной разработки *GitHub* имеет инструменты CI и тестирования сборки.
   * **Инструмент репортинга**: *Allure Framework* — создаёт детальные и понятные отчёты о выполнении тестов.

## 3. Перечень и описание возможных рисков при автоматизации.
   1. Отсутствие полной документации на тестируемый сервис заставляет строить предположения о поведении системы.
2. Автотесты в следствие необнаруженных дефектов в их коде могут предоставить недостоверные результаты.
3. Последующее изменение кода тестируемого приложения может сделать тесты непригодными к использования.
4. Необходимость написания кода автотеста, его поддержка влекут временные затраты.
5. Результат использования тестовых данных может отличаться от результатов использования реальных данных в промышленной эксплуатации.
6. Зависимость автотестов от окружения (ОС, используемые инструменты).
